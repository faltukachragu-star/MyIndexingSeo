OAuth 2.0 Setup Guide for the Tool Admin

This guide is for the person running the Rapid Indexer tool. You need to create Google API credentials so that users can sign in and use the tool with their own accounts.

1

Go to Google Cloud Credentials

  1. Go to the Google Cloud Platform Credentials page.
  2. Create a project if you don't have one, or select an existing one.
  3. Click + CREATE CREDENTIALS and select OAuth client ID.
2

Configure Consent Screen

If prompted, you must configure your consent screen first.

  1. Choose External and click Create.
  2. Fill in the required fields: App name (e.g., "Rapid Indexer"), User support email, and Developer contact information.
  3. Click Save and Continue through the "Scopes" and "Test Users" sections. You can leave them blank. Finally, go back to the dashboard.
3

Create the OAuth Client ID

  1. Go back to Credentials, click + CREATE CREDENTIALS → OAuth client ID again.
  2. For Application type, select Web application.
  3. Give it a name (e.g., "Rapid Indexer Web Client").
  4. Under Authorized redirect URIs, click + ADD URI. Enter the address of your application followed by /auth/callback.
    For local testing: http://127.0.0.1:8000/auth/callback
    For a live server: https://yourdomain.com/auth/callback
  5. Click Create.
4

Get Your Credentials

A popup will show your Client ID and Client Secret. Copy these. You will need to set them as environment variables to run the application.

You can also download the JSON file, which contains these values, for your records.

IMPORTANT: The person running the server must set these values. They are not for end-users of the tool.

export GOOGLE_CLIENT_ID="YOUR_CLIENT_ID_HERE"
export GOOGLE_CLIENT_SECRET="YOUR_CLIENT_SECRET_HERE"
export SECRET_KEY="A_RANDOM_STRONG_SECRET_STRING"
